§ 01

The problem

Small and mid-size agribusinesses in Kenya rely on manual ledgers and disconnected tools for inventory tracking, sales, and accounting. Stock discrepancies are common, financial reporting lags by weeks, and there is no real-time visibility into which products are moving or which customers are profitable.

Grade Agrovet needed a single platform to replace its existing patchwork of Excel sheets, paper receipt books, and phone-based credit tracking. Specifically, it needed to:

›Track stock movements across multiple product categories in real time
›Handle point-of-sale transactions with multiple payment methods (cash, M-Pesa, credit)
›Manage credit customers and accounts receivable with aging reports
›Generate financial reports for owners and the accountant
›Support multiple user roles with appropriate access control and audit trails
§ 02

The approach

I built Grade Agrovet as a modular monolith. The frontend is a Next.js + TypeScript application; the backend is a Flask + Python REST API; the database is a Supabase-managed PostgreSQL instance with Row-Level Security policies on every sensitive table.

The system is organized into discrete modules (inventory, POS, production, finance), each with its own routes, controllers, and schemas, but sharing common authentication and audit infrastructure. This is deliberately not a microservices architecture. For a solo developer working with a single client, a modular monolith is faster to ship, cheaper to host, and easier to debug.

§ 03

Key features

›Real-time inventory tracking with low-stock alerts
›Point-of-sale with cash, M-Pesa (Daraja API integration planned), and credit payment methods
›Credit customer management with aging reports and statements
›Role-based access control across four roles: Admin, Manager, Cashier, Accountant
›Audit logs for every financial transaction and stock movement
›Financial reports: P&L, cash flow, trial balance, top customers, top products
›Production tracking for value-added products
§ 04

Technical highlights

›Row-Level Security enforced at the database layer on every sensitive table; auth is not just a frontend concern
›Server-side data fetching with Next.js Server Components, minimizing client-side JS shipped
›Optimistic UI updates on POS to keep the cashier flow fast under unreliable connectivity
›Background jobs handled via a Render worker process
›Full security audit covering 12 specific findings: N+1 query patterns, missing pagination, missing rate limits, RLS gaps, input validation
›Migration of the entire system from Railway to Supabase + Render with zero downtime
§ 05

Challenges & solutions

Migration 403 errors.

During the Railway-to-Supabase migration, previously working endpoints returned 403 Forbidden. The error code by itself was misleading. The fix came from reading the response body, which revealed that pg_dump --no-acl had stripped the GRANT permissions from the dumped schema. Re-running schema GRANT statements after the import restored access immediately. I now treat the response body as the first thing to check on any 403, not the status code.

POS speed under poor connectivity.

Cashiers needed the system to feel instant even when the connection dropped to 2G or worse. I used optimistic UI updates so that scanning an item and adding it to a sale showed immediately; a background sync confirmed the database write. Clear UI states made it obvious when a transaction was pending versus confirmed.

Audit completeness without slowing down writes.

Logging every financial movement could have doubled write latency. I moved audit-log writes into PostgreSQL triggers on the relevant tables, so the application layer doesn't need to remember to log, and the writes happen in the same transaction as the original action.

§ 07

Lessons learned

→For Kenyan SMBs, M-Pesa integration is not a "nice-to-have"; it's the baseline expectation from day one.
→A modular monolith beats microservices for solo dev teams. Almost always.
→Always check the response body on a 4xx error. The status code is often a lie about what actually went wrong.
